# Contributing to T2 — Travel Terminal

Thank you for your interest in contributing to T2! This document provides guidelines for participating in the project.

## 🌟 How to Contribute

### 1. Report Issues

- Use [GitHub Issues](https://github.com/t2-travel-terminal/t2-travel-terminal/issues) to report bugs or request features.
- Search existing issues before opening a new one.
- Provide clear reproduction steps, expected behavior, and actual behavior.

### 2. Submit Pull Requests

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature-name`.
3. Make your changes with clear, focused commits.
4. Add or update tests where applicable.
5. Update documentation if your change affects user-facing behavior.
6. Push to your fork and open a Pull Request against the `main` branch.

### 3. Develop MCPs, Skills, or Agents

We encourage community contributions to the T2 ecosystem:

- **MCP Servers** — standardize access to hotel/travel data sources.
- **Skills** — reusable, composable capabilities (e.g., price sentiment, event impact).
- **Agents** — end-to-end autonomous workflows for revenue management, distribution, etc.

See `docs/` for extension guidelines and the Developer Center (coming soon).

## 📝 Code Standards

- Follow the existing code style.
- Write meaningful commit messages.
- Keep changes minimal and focused.
- Ensure all tests pass before submitting.

## 🏛️ Governance

T2 uses a transparent, merit-based governance model. Major architectural and licensing decisions are discussed publicly in GitHub Discussions and Issues.
